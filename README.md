# vhdl-sim

A command-line tool for running GHDL + cocotb simulations without manually maintaining Makefiles. Run `vhdl-sim` from anywhere in your project, pick your entity, architecture, and testbench interactively, and the tool handles the rest.

## Features

- Recursively scans `src/` for VHDL entities and architectures
- Discovers cocotb testbenches in `test/`
- Interactive arrow-key selection with smart testbench matching
- Auto-generates a cocotb-compatible Makefile and runs it immediately
- Re-run the last simulation instantly with `--rerun`
- Clean `Ctrl+C` handling — kills the entire GHDL process tree gracefully

## Requirements

- Python 3.10+
- GHDL
- cocotb

## Installation

Clone the repository and install into your project's virtual environment:

```bash
git clone https://github.com/yourname/vhdl-sim
cd vhdl-sim
pip install -e . --no-build-isolation
```

## Project structure

The tool expects this layout in your VHDL project:

```
your-project/
├── src/
│   ├── my_entity.vhd
│   └── subdir/
│       └── another_entity.vhd
└── test/
    ├── tb_my_entity.py
    └── tb_another_entity.py
```

- VHDL source files are discovered recursively under `src/`
- Testbenches must be named `tb_<entity_name>.py` and live in `test/`
- Simulation output (generated Makefile, waveforms, build artifacts) goes into `sim/`

## Usage

### Interactive run

From anywhere inside your VHDL project:

```bash
vhdl-sim
```

The tool will:

1. Scan `src/` for all entities and architectures
2. Prompt you to select an entity
3. Prompt you to select an architecture (if more than one exists)
4. Prompt you to select a testbench — matched testbenches appear first
5. Generate `sim/sim_run.mk` and run the simulation

### Re-run last simulation

```bash
vhdl-sim --rerun
```

Skips all prompts and re-runs the exact same entity/architecture/testbench combination as last time. Useful when iterating on a design or testbench.

### Stopping a simulation

Press `Ctrl+C` at any time. The tool sends `SIGTERM` to the entire GHDL process group, giving it 5 seconds to flush the waveform file cleanly before force-killing with `SIGKILL`.

## Generated Makefile

The tool writes `sim/sim_run.mk` before each run. You can inspect it to see exactly what is being compiled and simulated:

```makefile
# Auto-generated by vhdl-sim — do not edit by hand

SIM                    ?= ghdl
TOPLEVEL_LANG          ?= vhdl
EXTRA_ARGS             += --std=08
TOPLEVEL               := my_entity
ARCH                   := rtl
COCOTB_HDL_TIMEPRECISION = 1fs

VHDL_SOURCES += /abs/path/to/src/*.vhd
VHDL_SOURCES += /abs/path/to/src/subdir/*.vhd

SIM_ARGS               += --wave=$(TOPLEVEL).ghw
COCOTB_TEST_MODULES    := tb_my_entity
export PYTHONPATH      := /abs/path/to/test

include $(shell cocotb-config --makefiles)/Makefile.sim

clean::
    -@rm -f $(TOPLEVEL)
    -@rm -f e~$(TOPLEVEL).o
```

All source paths are absolute so the Makefile works correctly regardless of where `make` is invoked from.

## Waveforms

GHDL writes a `.ghw` waveform file to `sim/` after each run. Open it with GTKWave:

```bash
gtkwave sim/my_entity.ghw
```

## Gitignore

Add the following to your `.gitignore` to keep generated files out of version control:

```
sim/
```

## Notes

- VHDL 2008 (`--std=08`) is always used
- Entities with no architectures found are shown in the selection list but cannot be simulated
- Multiple architectures for the same entity trigger an additional selection prompt
- If no matching `tb_<entity>.py` exists, the tool warns and lets you pick from all available testbenches or exit
